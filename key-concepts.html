Key Concepts

Below are seven concepts tied to this project and its topic.

Baseline

An approved snapshot of requirements used for control.

Traceability

Bidirectional links among code, tests, requirements, and design.

Priority

Relative risk/value guiding implementation order (e.g., MoSCoW).

Volatility

Rate of requirement change; high volatility signals risk.

Change Request

Formal proposal to modify an approved artifact.

Impact Analysis

Risk/quality/schedule/cost evaluation before approval.

Configuration Management

Identify, audit, and version controlled items across releases.
